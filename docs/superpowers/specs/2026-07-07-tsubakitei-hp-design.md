# ツバキ亭HP 設計書 — Wix脱出（静的HTML + microCMS + GitHub Pages）

作成日: 2026-07-07
ステータス: 承認済み（YOSHI）

## 背景

顧客案件（P5・1件目）のツバキ亭HP（tsubakitei.tokyo）をWixで制作したが、以下の問題が発生した。

1. モバイル版が自動対応せず、PC版とスマホ版の二重編集が必要
2. 店主向けの編集画面（Wixダッシュボード）が想定より使いにくい
3. AI（Claude Code）での一括編集ができない

WordPress移行も検討したが、保守負担（サーバー・プラグイン更新・セキュリティ）と管理画面の複雑さから不採用。以下の構成を採用する。

## 採用構成

| レイヤー | 技術 | 理由 |
|---|---|---|
| サイト本体 | 静的HTML/CSS/最小JS（ビルド不要） | Claude Codeで最も編集しやすい。レスポンシブでモバイル自動対応 |
| メニューデータ | microCMS（無料枠）※Phase 2 | 店主がスマホで品名・価格・写真を直接編集できる |
| ホスティング | GitHub Pages（無料） | 既存GitHubアカウントで新規登録不要。git pushで即公開 |
| ドメイン | tsubakitei.tokyo（Wixで取得済み） | WixのDNS設定で外部に向けられる。移管不要 |

## 役割分担

- **店主さん**: microCMSのスマホ管理画面でメニュー品目（品名・価格・写真・注記・区分）を編集
- **YOSHIさん**: それ以外の全更新をClaude Code経由で実施（HTML直編集 → git push）

## サイト構造

```
/                  トップ（ブランド + 店舗カード3枚: ツバキ亭 / ツバキ&デリ(未リンク) / Coming soon）
/about/            ツバキ亭について
/menu/             メニュー（ランチ・ディナー）
/takeout/          テイクアウト
/gallery/          ギャラリー
/access/           アクセス
/reserve/          ご予約（電話 tel: + 食べログリンクのみ。予約システムは作らない）
/recruit/          採用情報
/deli/             2号店ツバキ&デリ（同構成・navから未リンク・noindex）
/deli/...          2号店配下ページ
```

- デザインは現行Wix版の世界観（クリーム地×赤・円形ロゴ・レトロ洋食）を再現
- 3号店が来たら同パターンで `/店舗名/` を追加するだけ

## メニュー表示（段階導入）

- **Phase 1（今回）**: `data/menu.json` から描画。データは公開中のtsubakitei.tokyoから全品スクレイプ
- **Phase 2（microCMS登録後）**: microCMSに「menu」APIを作成し、取得先を切替（クライアントサイドfetch・保存即反映）
  - フィールド: 品名 / 価格 / 写真 / 注記 / カテゴリ（ランチ・ディナー・テイクアウト）/ 店舗（ツバキ亭・デリ）/ 表示順
  - 店主向けにスマホ操作手順ページを用意

## 公開・切替手順

1. GitHub Pages仮URLで公開 → YOSHI・店主確認
2. Wixダッシュボード「DNSレコードを管理」でtsubakitei.tokyoをGitHub Pagesに向ける（A/CNAMEレコード）+ HTTPS有効化
3. DNS浸透後に切替完了（ダウンタイムほぼゼロ）
4. Wixプレミアムプランは次回更新日に解約（ドメイン契約のみ残す）

## 備考

- GitHub無料プランのPagesはpublicリポジトリが必要。2号店ページのソースは公開になるが、店自体は営業中・食べログ掲載済みのため実害なし。private化したい場合はGitHub Pro or Cloudflare Pagesへ切替（構成は同一）
- 写真の元データは後日受領 → それまでWix CDNの画像を使用
- とみ笑のGitHub移行は別件として後日検討（本件スコープ外）
